# Technical Roadmap

## Development Philosophy

Sylvaris should be developed through:
- vertical slices
- emotional prototypes
- iterative systems
- modular architecture

Do not attempt full-scale development immediately.

## Engine

Primary engine:
- Unreal Engine 5

## Programming Approach

Initial focus:
- Blueprints
- modular systems
- rapid iteration

Long-term:
- hybrid Blueprint + C++

## Production Tracks

### Track 1 — World Building
Lore, factions, creatures, systems, narrative.

### Track 2 — Visual Identity
Concept art, UI, architecture, atmosphere, biome style.

### Track 3 — Prototype Development
Movement, resonance systems, creature AI, traversal.

## First Prototype Goal

The first successful prototype should allow the player to:
- move through a biome
- use resonance vision
- encounter a creature
- interact emotionally
- trigger first fusion

If this feels magical, the project foundation succeeds.

## Prototype Development Order

1. Third-person controller
2. Traversal systems
3. Camera feel
4. Resonance vision
5. Spirit companion AI
6. Creature AI
7. Basic fusion system
8. Environmental interaction
9. Small biome slice

## Recommended Scope

Avoid early:
- giant open worlds
- advanced multiplayer
- massive creature rosters
- full economy simulation
- advanced crafting trees

Focus on:
- emotional quality
- traversal feel
- creature interaction
- atmosphere

## Creature Architecture

Creatures should use:
- modular data systems
- behavior trees
- AI perception
- animation state machines
- gameplay ability systems

## World Simulation Architecture

Use:
- region-based simulation
- state systems
- event escalation systems

Avoid:
- full global simulation

## Save System Requirements

Track:
- faction reputation
- creature relationships
- biome health
- event states
- world changes
- player progression

## Recommended Learning Path

### Phase 1
Unreal basics.

### Phase 2
Movement and traversal.

### Phase 3
Creature AI and interaction.

### Phase 4
Resonance systems.

### Phase 5
Vertical slice construction.

## Long-Term Possibilities

Potential future systems:
- AI dialogue
- procedural rumors
- dynamic broadcasts
- advanced ecosystem simulation
- multiplayer exploration
